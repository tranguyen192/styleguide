Brand Guidelines

Introduction

Travel Checklist is an interactive web application that helps you keep track of the three most important organizational aspects of your trip: What do I have to pack? What arrangements do I have to make? What errands do I have to run?

The follwing style guide contains a set of standards for creating and designing documents for this web application. It establishses and enforces a style to improve the brand communication. Consistency within a document and across multiple documents is essantial.

Embed Fonts

Copy this code into the head of your HTML document to embed the fonts into a webpage.

                        
<link href="https://fonts.googleapis.com/css?family=Josefin+Sans:400,700|Source+Sans+Pro:300,400" rel="stylesheet">

Or use this code to directly embed fonts into your CSS document.

                        
@import url('https://fonts.googleapis.com/css?family=Josefin+Sans:400,700|Source+Sans+Pro:300,400');

Use the following CSS rules to specify the font families.

                        
font-family: 'Josefin Sans', sans-serif;

// font-weight: 400;
// font-weight: 700;

font-family: 'Source Sans Pro', sans-serif;

// font-weight: 300;
// font-weight: 400;

Do not use other fonts. The fallback must always be a sans-serif font.

Logo

The word and design mark should only be used with white background. The height must never undershoot 200px. A minimum distance of 10% of the height of the logo to other elements must be respected in any case. The colors and fonts must not be changed. The logo must not be distorted in any way, single elements should not be moved and the proportions must not be changed.

Use SVG to directly embed the logo into your HTML document: download.

Figurative Mark

The figurative mark should only be used in appropriate contexts. The brand name must be communicated clearly at this point. The figurative mark should only be used with white background. The height must never undershoot 200px. A minimum distance of 10% of the height of the logo to other elements must be respected in any case. The colors must not be changed. The logo must not be distorted in any way, single elements may not be moved and proportions must not be changed.

CSS-Methodology

Class naming conventions

Use Hyphen Delimited Strings

Do not do this: 

                        
.headingsCode {
    border: 1px solid red;
}

Instead, do this: 

                        
.headings-code {
    width: 50vw;
}

It is consistent with the CSS property names.

                        
.headings-code {
    font-size: 0.8em;
    line-height: 1.8em;
}

BEM Naming Convention

B stands for block: The outermost parent element of the component is defined as the block. Class name selector is used preferred. Superfluous naming is to be avoided.

                        
.stick-man { 
}

E stands for elements: Inside of the component may be one or more children called elements. Elements are semantically tied to their block.

                        
.stick-man__head {  
}

.stick-man__arms {
}

.stick-man__feet {
}

M stands for modifiers: Either a block or element may have a variation signified by a modifier. Modifiers are used to change appearance, behavior or state.

                        
.stick-man--blue {
}

.stick-man--red {
}

Units

  • Font sizes are declared using pixels or em.
  • Height is set with view height or pixels.
  • Width is set with view width, pixels or percent.
  • Margin and padding are set with pixels.

Headings

When building a web page, having a clear page structure is important to organize the content of the page. This is best accomplished through the use of headings. Navigating through the <h1> and <h2> gives users an overview of a page and how its content is structured. Heading tags should be in order. That means an <h1> is followed by an <h2>, an <h2> is followed by a <h2> or <h3> and so on. PackIt uses four different headings.

Buttons

Buttons are clickable items used to perform an action. It should be used in situations where users might need to:

  • submit a form
  • begin a new task
  • specify a new or next step in a process
